Button

The Button prototype wraps a <button> or <input type="button"> element, and exposes the standard attributes of those elements as properties, which you can use with data binding, for example.

Button group

Putting buttons in a group is a matter of wrapping the buttons in an element and giving it a class.

Image

To add an image to a Button use CSS, either by setting a background image, or by using the :before and :after pseudo elements with the content property, as follows:

// HTML
<button class="btn settings" id="image">Settings</button>
// CSS
.settings:before {
    content: url(images/settings.png);
    vertical-align: middle;
    display: inline-block;
    padding-right: 0.5em;
}

Handling mouse/touch events on a Button

When a event occurs on a component (or any element) which has registered an event listener for the event type, the Montage event manager infers what function handler on the listener object to call. This topic is covered in detail in Event Handling.

For example, 

    

{
    "button2": {
        ...
        "listeners": [
            {
                "type": "action",
                "listener": {"@": "owner"}
            }
        ]
    }
}
This is equivalent to the following JavaScript: 
button2.addEventListener("action", owner);
The specified listener object (owner) object for a Button, implement a function with the following naming convention: 
"handle" + < button Identifier capitalized > + "Action"

Eg: for button with identifier = "button1": 

handleButton1Action: {
    value: function() {
        this.log("Button - button1 - clicked");
    }
}
